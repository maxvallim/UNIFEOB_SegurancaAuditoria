Configuração de Firewall no Cisco Packet Tracer

1. Criar o Cenário de Rede

Dispositivos Necessários: 2 PCs, 1 roteador, 1 switch e 1 servidor.

Conexões:

  • Conecte os PCs ao switch usando cabos Ethernet.
  • Conecte o switch ao roteador.
  • Conecte o servidor diretamente ao roteador.

Configuração de IPs e Máscaras

Dispositivo IP Máscara Gateway
PC1 192.168.1.2 255.255.255.0 192.168.1.1
PC2 192.168.1.3 255.255.255.0 192.168.1.1
Servidor 192.168.2.2 255.255.255.0 192.168.2.1
Roteador (Fa0/0) 192.168.1.1 255.255.255.0 -
Roteador (Fa0/1) 192.168.2.1 255.255.255.0 -

2. Configurar o Firewall no Servidor

Acessar o Servidor:

  1. Clique no servidor no Cisco Packet Tracer.
  2. Vá até a aba Config e selecione Firewall.

Definir Regras do Firewall

Serviço Porta(s) Remote IP Remote Wildcard Mask Ação Observação
HTTP 80 192.168.1.0 0.0.0.255 Permitir Permite acesso ao servidor web.
HTTPS 443 192.168.1.0 0.0.0.255 Permitir Permite tráfego seguro.
FTP 20, 21 192.168.1.0 0.0.0.255 Permitir Permite transferência de arquivos.
ICMP (ping) - 192.168.1.0 0.0.0.255 Bloquear Bloqueia testes de conectividade.
DNS 53 192.168.1.0 0.0.0.255 Permitir Permite consultas de nomes de domínio.

Certifique-se de clicar no botão Salvar para aplicar as regras configuradas.

3. Testar o Cenário

Teste de Conectividade

Use o comando ping nos PCs para verificar:

  • Teste o endereço IP do servidor (192.168.2.2).
  • Certifique-se de que o ICMP está bloqueado corretamente.

Teste de Serviços

  • HTTP/HTTPS: Abra o navegador em um dos PCs e acesse http://192.168.2.2.
  • FTP: Use um cliente FTP para conectar ao servidor no IP 192.168.2.2.
  • DNS: Realize consultas DNS para verificar funcionalidade.

Revise as regras do firewall e as configurações de IPs/máscaras caso algum teste falhe.
